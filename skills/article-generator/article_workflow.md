# 文章生成工作流详解

## 完整流程

```
输入解析 → 检索判断 → 材料整理 → 大纲生成 → 标题生成 → 正文生成 → 自检 → 转发文案 → 替补标题 → 写入 Excel
```

## 各步骤详细说明

### 步骤 1：输入解析（input_parser.py）

**输入**：自然语言文本 或 Excel/CSV 文件

**处理**：
- 按 `；` / `;` / 换行符 分割多个测例
- 从每个测例中提取：
  - 主题（关键词：以…为主题 / 关于… / 写一篇…文章）
  - 字数（正则：`\d+字`）
  - 图片数（正则：`\d+[个张]图片占位符` / `添加\d+[个张]图片`）
  - 风格要求（关键词：风格 / 受众 / 平台 / 营销 / 科普）
- 未提取到的字段使用默认值

**输出**：`List[TestCase]`

### 步骤 2：检索判断（material_retriever.py）

**判断规则**：
- ✅ 需要检索：涉及近期事件、政策、市场数据、行业趋势、具体公司、产品发布、技术更新
- ❌ 不需要检索：常识性、经验型、观点型主题

**检索关键词触发列表**：
- 最新、近期、今年、本月、上月
- 政策、数据、报告、趋势、市场
- 发布、上市、更新、版本、融资、收购

**当前实现**：mock 检索结果，预留搜索 API 接口

**输出**：`MaterialResult(need_retrieval, materials_text, retrieved)`

### 步骤 3：生成大纲

**要求**：
- 结构清晰：引言 → 主体（2-4 个分论点）→ 结论
- 与主题紧密相关
- 每个分论点下注明大约需要多少字

**LLM 提示词结构**：
```
你是一个专业的内容策划。请为主题"{topic}"生成一篇文章大纲。
目标字数：{word_count}字
风格：{style}
{materials_section}

要求：
- 结构清晰，包含引言、主体（分论点）、结论
- 每个部分标注预估字数
- 标注图片占位符的建议插入位置（用 {IMAGE_N} 标记）
```

### 步骤 4：生成标题

**标题要求**：
- 有吸引力、抓人眼球、利于传播
- 但不能标题党、不能偏离主题、不能夸大事实、不能虚假承诺

**LLM 提示词结构**：
```
基于以下大纲为主题"{topic}"生成一个文章标题。
大纲：{outline}

要求：
- 有吸引力和传播力
- 不标题党、不夸大、不虚假承诺
- 紧扣主题
```

### 步骤 5：生成正文

**正文要求**：
- 围绕主题展开，结构清晰，逻辑完整
- 与大纲一致
- 避免流水账、重复表达、过度口号化、明显 AI 腔
- 接近目标字数（±10%）
- 在合适位置插入指定数量的图片占位符

**图片占位符格式**：严格使用 `![AI配图中...](图片n路径)` 格式，n 从 0 开始连续递增

**LLM 提示词结构**：
```
基于以下大纲生成一篇完整的文章正文。
主题：{topic}
大纲：{outline}
目标字数：{word_count}字
图片占位符数量：{image_count}个
风格：{style}

要求：
1. 正文与大纲结构一致
2. 文章顶部第一行为 # {标题} 格式的标题
3. 首段迅速进入主题，有冲突感或价值感
4. 避免流水账、空话、AI 腔
5. 在合适位置插入 {image_count} 个图片占位符，
   格式严格为：![AI配图中...](图片n路径)
   n 从 0 到 {image_count-1}，分散插入
6. 字数控制在 {word_count} ±10% 范围内（不含标题）
```

### 步骤 6：自检（article_checker.py）

**检查项目**：
| 检查项 | 规则 |
|--------|------|
| 主题匹配 | 正文是否围绕指定主题 |
| 字数 | 是否在目标字数 ±10% 内 |
| 图片占位符数量 | 是否等于用户要求数量 |
| 图片占位符编号 | 是否从 0 开始连续递增 |
| 图片占位符格式 | 是否严格 `AI配图中...` |
| 大纲一致性 | 正文结构是否符合大纲 |
| 标题质量 | 是否抓人但不跑题 |
| 首段吸引力 | 是否快速进入主题 |
| 重复/空话 | 是否有明显重复或空泛 |

**自动修复**：
- 检查失败 → 自动修复正文 → 再次检查
- 最多修复 2 次
- 仍失败 → 记录日志，输出当前最好版本

### 步骤 7：生成转发文案

**要求**：2-3 句简洁转发文案

**LLM 提示词结构**：
```
基于以下文章，生成 2-3 句简洁的转发文案，适合在朋友圈或社群分享。
主题：{topic}
正文摘要：{body_summary}
```

### 步骤 8：生成替补标题

**要求**：3 个不同风格的替补标题

**LLM 提示词结构**：
```
基于以下文章，生成 3 个不同风格的替补标题。
主标题：{title}
主题：{topic}

要求：每个标题风格不同（如疑问式、数据式、冲突式）
```

### 步骤 9：写入 Excel

- 创建 `article_outputs/` 目录（如不存在）
- 写入 `article_outputs/generated_articles.xlsx`
- 严格四列：主题、文章正文、标题、转发文案
- 各列内容：
  - **主题**：原始主题文本
  - **文章正文**：Markdown 格式，顶部带 `# {标题}`
  - **标题**：主标题 + 3 个替补标题
  - **转发文案**：仅 2-3 句转发文案（不含替补标题）
- 标题列格式：
  ```
  主标题：xxx

  替补标题：
  1. xxx
  2. xxx
  3. xxx
  ```
- 转发文案列格式：
  ```
  1. xxx
  2. xxx
  ```
