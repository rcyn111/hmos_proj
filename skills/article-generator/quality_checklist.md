# 文章质量检查清单

每篇文章生成后，必须逐项检查以下所有内容。

## 一、元数据检查

- [ ] 主题是否与用户输入一致
- [ ] 字数是否在目标值 ±10% 范围内
- [ ] 图片占位符数量是否正确
- [ ] 写作风格是否符合要求

## 二、图片占位符检查

- [ ] 格式严格为 `![AI配图中...](图片n路径)`（每行一个）
- [ ] alt 文本严格为 `AI配图中...`
- [ ] 路径严格为 `图片n路径`，n 从 0 开始连续递增
- [ ] 没有多余字符（没有空格、括号嵌套、中文数字等）
- [ ] 分散在正文合适位置，未全部堆在开头或结尾
- [ ] 如果要求 0 张图片，正文中完全无占位符

### 错误格式示例（必须拒绝）

| 错误写法 | 问题 |
|----------|------|
| `图片一：xxx` | 使用了中文数字 |
| `图片 1：xxx` | 不是 Markdown 图片语法 |
| `image0：xxx` | 使用了英文 |
| `AI配图中...` | 缺少 Markdown 图片语法（没有 ![]()） |
| `![AI配图中...](图片0)` | 路径缺少"路径"二字 |
| `![AI配图中...](图片1路径)` | 编号应从 0 开始 |
| `![配图中...](图片0路径)` | alt 文本缺少 "AI" |

## 三、标题检查

- [ ] 有吸引力，能抓人眼球
- [ ] 利于传播（适合分享到朋友圈/群聊）
- [ ] 未标题党（不夸大、不虚假承诺）
- [ ] 未偏离主题
- [ ] 与正文内容一致

## 四、首段检查

- [ ] 迅速进入主题（前 3 句内）
- [ ] 有冲突感、问题感或价值感
- [ ] 能让读者愿意继续读
- [ ] 未使用"在当今社会""随着时代发展"等空泛套话

## 五、正文检查

- [ ] 结构清晰，分点合理
- [ ] 逻辑完整，论证充分
- [ ] 与大纲一致
- [ ] 无流水账式堆砌
- [ ] 无明显重复表达
- [ ] 无过度口号化（如连续多个感叹号、空洞的号召）
- [ ] 无明显 AI 腔（如"综上所述""值得注意的是""不可否认"等高频使用）
- [ ] 无事实性夸大或编造

## 六、转发文案检查

- [ ] 2-3 句，简洁有力
- [ ] 适合社交媒体分享
- [ ] 能激发读者点击/阅读兴趣

## 七、替补标题检查

- [ ] 共 3 个
- [ ] 风格各不相同（如疑问式、数据式、冲突式、解决方案式）
- [ ] 每个都紧扣主题
- [ ] 与主标题不重复

## 八、最终 Excel 检查

- [ ] 只包含四列：主题、文章正文、标题、转发文案
- [ ] 标题列包含主标题 + 3 个替补标题
- [ ] 文章正文列包含 Markdown 格式正文（顶部有 `# {标题}`）
- [ ] 转发文案列仅包含 2-3 句转发文案（不含替补标题）
- [ ] 所有行数据完整，无空值
- [ ] 文件名正确：`article_outputs/generated_articles.xlsx`

## 自动修复机制

如果任何检查项失败：
1. 根据失败项生成修复指令
2. 调用 LLM 重新生成/修复正文
3. 再次执行完整检查
4. 最多修复 2 次
5. 第 3 次仍失败 → 记录失败原因，输出当前最好版本
