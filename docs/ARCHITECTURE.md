# 架构说明 — HarmonyOS Dev Claw

> 本文档说明 HarmonyOS Dev Claw 的整体架构设计。
> 面向想理解 Claw 如何运作的新手和贡献者。

## 总体架构

HarmonyOS Dev Claw 不是一个跑在后台的服务，而是一套**运行在 Claude Code 中的规则 + Skill + 知识库体系**。

## 核心设计原则

### 1. 文件驱动，零依赖

- 所有配置和知识都存为 Markdown / YAML / 文本文件。
- 不需要数据库、不需要后端服务、不需要向量数据库。
- 版本管理直接用 Git。

### 2. Skill 是第一公民

- 每个 skill 是一个独立目录，包含一个 `SKILL.md`。
- SKILL.md 完整定义该 skill 的：
  - 使用场景（什么时候调用）
  - 输入输出格式
  - 执行步骤
  - 成功/失败标准
  - 复盘要求
- Claude Code 读取 SKILL.md 后按其中的步骤执行。

### 3. 闭环进化

```
执行 → 记录(runs/) → 分析 → 优化 skill(.claude/skills/) → 记录变更(memory/) → 下次执行更准
```

### 4. 新手优先

所有设计决策先问："新手能理解吗？"

## 目录结构设计逻辑

| 目录 | 定位 | 谁来写 | 什么时候更新 |
|------|------|--------|-------------|
| `CLAUDE.md` | 最高规则 | 人工 | 项目规则变更时 |
| `config/` | 配置 | 人工 + Claw | 配置变更时 |
| `docs/` | 静态文档 | 人工 | 产品/架构变更时 |
| `knowledge/` | 动态知识库 | Claw 自动 | 每次学到新知识时 |
| `.claude/skills/` | Skill 定义 | 人工 + Claw | Skill 进化时 |
| `prompts/` | 提示词模板 | 人工 | 需要新提示词时 |
| `runs/` | 执行复盘 | Claw 自动 | 每次 skill 执行后 |
| `memory/` | 长期记忆 | Claw 自动 | 发现偏好/决策/进化时 |
| `scripts/` | 辅助脚本 | 人工 | 需要自动化脚本时 |
| `projs/` | 用户项目 | 用户 + Claw | 开发过程中 |

## 数据流

```
用户输入
  │
  ├─→ CLAUDE.md（读取规则）
  │
  ├─→ knowledge/（查询已有知识）
  │
  ├─→ .claude/skills/（匹配 skill）
  │
  ├─→ WebSearch / WebFetch（搜索新知识）
  │
  ├─→ 执行 skill（生成或修改代码）
  │     │
  │     ├─→ projs/（写入代码文件）
  │     │
  │     └─→ runs/（生成复盘记录）
  │           │
  │           └─→ harmony-skill-evolver（分析优化）
  │                 │
  │                 ├─→ .claude/skills/（更新 SKILL.md）
  │                 └─→ memory/skill_evolution_log.md（记录变更）
  │
  └─→ knowledge/（沉淀新知识）
```

## 安全架构

```
┌──────────────────────────────────────┐
│           信任边界                      │
│                                      │
│  ┌────────────────────────────┐      │
│  │  项目目录内（允许访问）         │      │
│  │  /Users/.../hmos_proj/     │      │
│  │                            │      │
│  │  ✅ 读写所有文件              │      │
│  │  ✅ 创建子目录               │      │
│  │  ✅ Web 搜索                │      │
│  │                            │      │
│  │  ⚠️ 删除需确认              │      │
│  │  ⚠️ 大改需确认              │      │
│  │  ❌ 禁止存储密钥             │      │
│  └────────────────────────────┘      │
│                                      │
│  ┌────────────────────────────┐      │
│  │  项目目录外（禁止访问）         │      │
│  │                            │      │
│  │  ❌ 禁止读取                 │      │
│  │  ❌ 禁止写入                 │      │
│  │  ❌ 禁止执行                 │      │
│  └────────────────────────────┘      │
│                                      │
│  API key 仅存在于环境变量中               │
│  项目文件中只有 .example 不含真实 key      │
└──────────────────────────────────────┘
```
